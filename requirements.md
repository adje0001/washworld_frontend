# WashWorld Requirements

FRONTEND REQUIREMENTS
The project does not need every small detail, but everytime we do one of the mentioned below, please make a comment to point it out.

• Component-based architecture
• State management using useState and props
• useEffect for side effects
• At least one custom hook
• Data fetching using fetch and TanStack Query with loading and error handling
• Forms with validation
• REST API integration using correct HTTP methods
• JWT authentication (store token and send in Authorization header)
• Conditional rendering (loading, error, empty states)
• E2E Testing with Cypress
• Search or filtering functionality
• Optimistic UI updates

BACKEND REQUIREMENTS
The backend must:
be a REST API

Must validate everything that comes from the user
Return JSON whenever the front-end requires it
Implement JWT authentication
Password must be hashed when signing up

HTTP status codes must be used

HTTP methods must be properly applied

An email is sent to the user “welcoming” upon signup

Forgot password feature must exist and an email is sent to the user
